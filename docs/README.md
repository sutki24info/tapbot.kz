## 🎯 ПРИНЦИПЫ ДОКУМЕНТАЦИИ

### Основанные на мировых стандартах:

-  **Иерархическая нумерация** (ISO/IEC подход)

-  **Модульность** (TOGAF Enterprise Architecture)

-  **Жизненный цикл** (ITIL Service Management)

-  **Точечные обновления** (RFC Engineering Approach)

### Адаптированные для стартапа:

-  **Практичность** - без бюрократии

-  **Масштабируемость** - от MVP до enterprise

-  **Делегирование** - понятно новичкам

-  **Agile** - быстрые изменения

-  **Прозрачность** - публичные планы для команды

-  **Конфиденциальность** - стратегические решения в приватной зоне

## 🔄 ПРАВИЛА ОБНОВЛЕНИЯ ДОКУМЕНТАЦИИ

### Публичная документация (GitHub):

```
Формат: "Обновить раздел XX.YY.ZZ"
Пример: "Обновить разделы 02.01.01 и 03.02.03"
Результат: Только указанные разделы переписываются
Доступ: Команда через Gramax, AI через project_knowledge_search
```

### Приватная документация (AI база знаний):

```
Формат: "Обновить раздел AI-XX.YY"
Пример: "Обновить разделы AI-02.03 и AI-07.01"
Результат: Экономия символов генерации, точечные изменения
Управление: Локальное редактирование + ручная загрузка в базу знаний
```

### Типы изменений:

-  **MINOR** - изменение содержимого раздела без структуры

-  **MAJOR** - добавление/удаление подразделов

-  **CRITICAL** - изменение основной структуры документа

### Процедура обновления:

1. **Определить затронутые разделы** по номерам

2. **Указать тип изменения** (MINOR/MAJOR/CRITICAL)

3. **Обновить только нужные разделы**

4. **Обновить версию и changelog**

5. **Синхронизировать с соответствующей системой** (GitHub или AI база знаний)
